# Plan: Phase 02 — Decision Engine

Implement deterministic nudge selection with three evaluator functions, confidence tiers, fatigue controls, and escalation behavior in a single `server/app/engine.py` module. Add missing `meal_logged` seed signal so all three scenarios are testable.

## Decisions

- **Single `engine.py` module**: All evaluator functions, candidate model, fatigue logic, selection, and persistence live in one file. Keeps the prototype reviewable without premature packaging.
- **Seed data addition**: Added `meal_logged` signal for `member_meal_01` — `carbs_g: 72`, created ~6 hours ago — so `check_meal_goal_mismatch` can trigger against seeded data.
- **`content` left `None`**: Nudge `content` field is not populated at creation. Phase 6 (LLM phrasing) fills it. `explanation_basis` carries the factual summary downstream.
- **`explanation_basis` is terse**: e.g. `"Pasta Carbonara logged 72g carbs; goal is low_carb (<60g)"` — enough for coach view and later LLM phrasing input, not member-facing copy.
- **`evaluate_member()` commits**: The engine commits after creating a nudge. Acceptable for Phase 2 where the engine is the only writer; revisit if Phase 3 needs transactional control across multiple operations.
- **`latest_signal_ts` tie-break**: Added to `NudgeCandidate` model per spec requirement ("tie-break on most recent matching signal"). Two-pass stable sort: recency DESC → (priority ASC, confidence DESC).

## Phase B: Candidate model

2. `NudgeCandidate` Pydantic model with fields:
   - `nudge_type`, `matched_reason`, `explanation_basis`, `confidence`
   - `escalation_recommended`, `source_signal_ids`, `priority`, `latest_signal_ts`

## Phase C: Evaluator functions

Each evaluator takes `(conn, member_id)` and returns `NudgeCandidate | None`.

3. **`check_meal_goal_mismatch`**:
   - Query `members.goal_type` → skip if not `low_carb`
   - Query most recent `meal_logged` signal in last 24h
   - Check `carbs_g >= 60` → candidate with confidence `0.86`, priority `2`

4. **`check_missing_weight_log`**:
   - Cutoff = start of current UTC day minus 4 days
   - Query for any `weight_logged` signal since cutoff
   - If none found → candidate with confidence `0.68`, priority `3`, `source_signal_ids=[]`

5. **`check_support_risk`**:
   - Query `mood_logged` with `mood == "low"` in last 3 days
   - Count `dismiss` actions on member's nudges in last 7 days
   - If low mood AND dismissals ≥ 2 → candidate with confidence `0.42`, `escalation_recommended=True`, priority `1`

## Phase D: Fatigue and deduplication

6. **`get_active_nudge(conn, member_id)`**: Returns existing active nudge row (idempotent read path).

7. **`_check_cooldown(conn, member_id, nudge_type)`**: Returns True if same `nudge_type` had `act_now` or `dismiss` action within 24 hours.

8. **`_count_today_nudges(conn, member_id)`**: Counts auto-delivered nudges (`active`, `acted`, `dismissed`) created today UTC.

9. **`_apply_fatigue(conn, member_id, candidates)`**: Filters candidates by cooldown and daily cap (max 2). `support_risk` bypasses both.

## Phase E: Selection

10. **`select_nudge(conn, member_id)`**: Runs all evaluators → collects candidates → applies fatigue → sorts by priority ASC, confidence DESC, latest signal DESC → returns top candidate or None.

## Phase F: Persistence and escalation

11. **`create_nudge_from_candidate(conn, member_id, candidate)`**:
    - If `confidence < 0.50` or `escalation_recommended`:
      - Persist nudge with `status='escalated'`
      - Create `escalations` row: `source='rule_engine'`, `status='open'`
      - Return `{"state": "escalated", "nudge_id": ..., "escalation_id": ...}`
    - Otherwise:
      - Persist nudge with `status='active'`, `delivered_at` set
      - Return `{"state": "active", "nudge": {...}}`

12. **`evaluate_member(conn, member_id)`** — top-level entry point:
    - Existing active nudge → return it (idempotent, no re-evaluation)
    - No active nudge → `select_nudge()` → `create_nudge_from_candidate()` if match
    - No candidate → `{"state": "no_nudge"}`

## Verification

## Constants

All thresholds are explicit module-level constants in `engine.py`:

| Constant                   | Value  | Purpose                                           |
| -------------------------- | ------ | ------------------------------------------------- |
| `CONFIDENCE_LOW_THRESHOLD` | `0.50` | Below this → escalate, don't auto-deliver         |
| `COOLDOWN_HOURS`           | `24`   | Same nudge_type blocked after act/dismiss         |
| `DAILY_CAP`                | `2`    | Max auto-delivered nudges per member per UTC day  |
| `MISSING_WEIGHT_DAYS`      | `4`    | No weight signal in this many days triggers nudge |
| `MEAL_CARB_THRESHOLD`      | `60`   | Carbs ≥ this for low_carb goal triggers nudge     |
| `MEAL_LOOKBACK_HOURS`      | `24`   | How far back to look for meal signals             |
| `MOOD_LOOKBACK_DAYS`       | `3`    | How far back to look for low mood                 |
| `DISMISS_LOOKBACK_DAYS`    | `7`    | How far back to count dismissals                  |
